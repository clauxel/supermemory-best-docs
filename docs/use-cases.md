# Use Cases - Memory Hygiene Audit

Memory Hygiene Audit scans AI agent memory and repo instructions for stale facts, context bloat, unsafe retention, contradictions, and poisoning risk.

Stop old repo notes from steering new sessions toward stale build or deployment advice.

## AGENTS.md governance

Keep multi-agent workspace instructions scoped, current, and safe.

## MCP memory poisoning review

Detect tool-output or external-text instructions that should not become durable memory.

## Retention policy rollout

Create a repeatable rule set for what should be remembered, summarized, or deleted.
