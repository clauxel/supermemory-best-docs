# Guides - Memory Hygiene Audit

Memory Hygiene Audit scans AI agent memory and repo instructions for stale facts, context bloat, unsafe retention, contradictions, and poisoning risk.

## Classify memory sources

Mark each note as repo fact, user preference, temporary incident, external content, tool output, or policy rule.

## Trim context bloat

Delete expired incidents, merge duplicate rules, shorten repeated notes, and keep only decision-relevant facts.

## Review poisoning flags

Treat instructions to ignore rules, preserve secrets, or trust untrusted text as high-risk until a human approves.
